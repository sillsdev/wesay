Welcome to WeSay 0.5.x-Development

You have installed a test version of WeSay.  This is a team effort, and we really appreciate your willingness to give WeSay a spin and send us feedback. 

About this version

WeSay comes in two flavors: “stable” (even numbered ones) and “development” (odd numbered ones).   The even versions receive critical bug fixes, but not new features or anything risky.

If crashes or other bugs would be a major problem for you, please uninstall this version and get our most recent “even” numbered one.  If you are evaluating WeSay’s capabilities or are willing to risk the occasional bug, this is the version for you.  This is where we are currently adding new abilities in response to your requests, and we need people to help ensure that it can become stable, too. Eventually, we will retire 0.4 and 0.5 will be relabeled “0.6 - Stable”. Note that at times we post new “odd” versions almost daily.  If you report a bug via issues@wesay.org, we can often times fix it and provide a new release within hours.  So let us know if you’re in a hurry for a fix.

What’s New in 0.5?

Here’s what we’ve added, so far, to 0.5.  More improvements are planned.

Enter Example Translations in Add Missing Example Sentences Task

In the “Add Missing Example Sentences” task, You can now enable a place for immediately collecting a translation of the example.  (This was always possible in the Browse & Edit view, of course).

Add translation in Gather By Semantic Domain Task

You can now turn on an option to show a “meaning” field in the Gather By Semantic Domains task.   Be warned, this has some problematic aspects.  For example, if you later click on a word you previously entered, and change its spelling, you will now get a new word, and the old spelling will remain.  Some of this may seem like a “bug” to a human, but it’s hard for the program to safely guess what the right thing is to do, and the “WeSay way” is to avoid asking the user questions.  We welcome your ideas on this. In the meantime, we leave it to you to decide if the benefits outweigh the problems.

Semantic Domain Numbers

By popular demand, numbers have returned, everywhere we show Semantic Domain labels.  This is a good time to note that if you want to add semantic domains in the Browse & Edit view, you can do that by typing in the number of the domain, or search for an applicable domain by typing in relevant words.

Linux

We’ve got WeSay running on Ubuntu Linux, so if you would like to help us test that version too, let us know.  We even have SD chips which boot an OLPC XO into Ubuntu so you can run WeSay in indestructible 5 watt bliss.  Some features are as yet unavailable on Linux, including USB-stick backup,  email, audio features, and pdf dictionary making.

Peace of Mind

An important new thing we’re adding in 0.5 is automated incremental backup.  The way this works is that you put some kind of memory stick into the laptop (like the memory your digital camera uses), then tell WeSay where to find it.  You just leave that memory chip in, permanently.  WeSay will silently keep changes there, in a way that you can always “roll-back” if things go very wrong.  This is the first feature that uses the Chorus collaboration component SIL is building.  To enable this and forthcoming Chorus features, you need to install a package called “Mercurial”. On Windows, download and install TortoiseHG.

One click to Create PDF Dictionary

A PDF dictionary is now a mouse-click away, once you install princexml.  (This feature is Windows only, temporarily). Please, >let us know if you use this feature.

Open With Lexique Pro

If you need more some more than the basics, WeSay now provides a button for opening your dictionary in Lexique Pro (version 3) (Windows only).  See http://lexiquepro.com/test.htm

Audio Recording and Playback

You can now add audio recording and playback to any text field.  Simply create a new writing system, and mark it “Audio”.  Then tick this writing system for any fields (e.g. Word or Example) where you want to enable users to speak the text, as well as write it.  Audio files are created in the new “audio” folder of the WeSay project.  (Temporarily Windows only).  Please, let us know if you use this feature.

Where was I?

WeSay now remembers where you were in the dictionary browse & edit.  It also remembers any adjustments you made to change the size of parts of the screen.

What Now?

Get your head around WeSay… it’s really different.  It’s aimed at empowering a very different user than you are, and approaches the task of dictionary making in a new, super-task-oriented way.  Using the provided samples, play around with both WeSay and the WeSay Configuration Tool.  To do this, go to your Windows Start menu and find “WeSay”.  Choose one of the icons there which open the sample project.
